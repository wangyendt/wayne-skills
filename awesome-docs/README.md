# Awesome Docs Skill

这个目录是一个可迁移的 skill，用于把工程文档变成 AI 执行和人类理解之间的稳定接口。

核心能力：

- 默认使用项目根目录下的 `docs/`，也允许用户指定文档路径。
- 自动维护 `plan/`、`experiment/`、`knowhow/`、`roadmap/`。
- 根据用户意图把文档记录到合适目录。
- 新内容优先创建 `YYYYMMDD_标题.md`，明确指定旧文档时才更新旧文档。
- 写作风格强调可读性、结构化、简洁、核心数据和结论。
- 当用户要求保存脚本命令或写入常用命令时，维护 `docs/常用命令.txt`，命令必须是单行。

使用方式：

1. 将整个 `awesome-docs-skill/` 目录复制到目标系统支持的 skills 目录，或上传到你的 skill 分发位置。
2. 目标系统读取 `SKILL.md` 后，即可按其中约定维护工程文档。

主要文件：

- `SKILL.md`：skill 主体定义和工作流。
